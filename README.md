## [0x00] What this is (and what it isn't)

| | stryke-arrow | **stryke-parquet** |
|---|---|---|
| Surface | Full Arrow data pipeline: parquet / IPC / CSV / JSON / Feather read+write, conversion, DataFrame bridge | **Parquet-only diagnostics** ‚ÄĒ inspect, schema, head/tail, row-group breakdown, per-column stats, recompress |
| Best for | "I want to read or write parquet data and do things with it" | "I have a parquet file and I want to understand it" |
| Reference | arrow-rs | `pqrs` / `parquet-tools` |
| Binary size | ~8.5 MB | ~6 MB |
| Output | NDJSON / columnar / Arrow IPC | NDJSON / one-line JSON |

The two packages share the same `parquet` + `arrow` crates underneath but
serve different user intents. Use stryke-arrow when parquet is one stop
in a data pipeline; use stryke-parquet when parquet itself is the
artifact you're investigating.

## [0x02] Quick start

```stryke
use Parquet

# Footer-only ‚ÄĒ fast, no row scan.
p to_json Parquet::inspect "events.parquet"
p Parquet::count   "events.parquet"
p to_json Parquet::schema "events.parquet"

# Per-row-group breakdown.
val @rgs = Parquet::rowgroups "events.parquet"
for val $rg (@rgs) {
    p "rg $rg->{ordinal}: $rg->{num_rows} rows, $rg->{total_compressed_size} bytes"
}

# Per-column stats (aggregated across row groups).
Parquet::stats "events.parquet" |> ep
Parquet::stats "events.parquet", column => "user_id"

# Peek at rows.
val @first = Parquet::head "events.parquet", n => 20
val @last  = Parquet::tail "events.parquet", n => 5
val @cols  = Parquet::head "events.parquet", n => 10, columns => ["user_id", "ts"]

# Stream every row (no full-result buffering ‚ÄĒ for big files).
Parquet::stream "events.parquet",
    callback => fn { process _ }

# Convert / recompress.
val $csv = Parquet::to_csv "events.parquet"                       # ‚Üí scalar
Parquet::to_csv "events.parquet", output => "events.csv"          # ‚Üí file
Parquet::compress "events.parquet", "events.zst.parquet",
                  codec => "zstd"                                  # recompress
```

## [0x03] FFI layer

Each `Parquet::*` wrapper builds a JSON args dict and calls a sibling
`parquet__*` symbol resolved out of `libstryke_parquet.{dylib,so}`. The
cdylib is dlopened in-process on first `use Parquet` (via stryke's
`pkg::commands::try_load_ffi_for` resolver hook). Its exports span
inspection (`version`, `inspect`, `schema`, `dtypes`, `count`, `rowgroups`,
`stats`, `metadata`), row read (`head`, `tail`, `to_json`, `to_csv`,
`to_ndjson`), row predicates / reshaping (`filter`, `where_count`,
`filter_to`, `distinct`, `sort`, `column`, `sum`, `describe`, `group_by`,
`random_sample`), conversion (`from_csv`, `from_json`, `write`,
`write_partitioned`, `compress`, `repartition`, `merge`, `select`), and
diagnostics (`validate`, `column_chunk_stats`, `size_report`, `null_summary`,
`encoding_summary`, `row_group_summary`, `sample`, `features`). The
authoritative list is `[ffi].exports` in `stryke.toml`.

Stateless package ‚ÄĒ parquet operations are file transforms; no
process-level cache.

## [0x04] API reference

```stryke
Parquet::inspect    $path ‚Üí \%info
Parquet::schema     $path ‚Üí { fields, num_fields }
Parquet::count      $path ‚Üí $n
Parquet::rowgroups  $path ‚Üí @rgs
Parquet::row_group_summary $path ‚Üí { num_row_groups, total_rows, total_compressed_size, rows_per_group:{min,max,mean}, compressed_bytes_per_group:{min,max,mean} }   # footer-only sizing rollup
Parquet::stats      $path, %opts ‚Üí @stats         # opts: column
Parquet::head       $path, %opts ‚Üí @rows          # opts: n, columns
Parquet::slice      $path, %opts ‚Üí @rows          # opts: offset, length (to end if omitted), columns ‚ÄĒ offset window
Parquet::tail       $path, %opts ‚Üí @rows
Parquet::reverse    $path, %opts ‚Üí @rows          # every row in reverse file order (newest-first); opts: columns
Parquet::gather     $path, \@indices, %opts ‚Üí @rows  # rows by explicit 0-based index list (arbitrary order, repeats OK, out-of-range dies); opts: columns
Parquet::top_k      $path, $column, $k, %opts ‚Üí @rows # the k rows with the largest values in $column (polars top_k); opts: descending => 0 for the smallest (bottom_k); nulls sort last, k caps at row count
Parquet::value_counts $path, $column ‚Üí @rows         # { value, count } per distinct value in $column (pandas/polars value_counts); sorted by count desc then value asc
Parquet::to_json    $path, %opts ‚Üí @rows
Parquet::stream     $path, %opts ‚Üí $count         # callback per row
Parquet::to_csv     $path, %opts ‚Üí $csv | $path   # opts: output, columns
Parquet::compress   $src, $dst, %opts ‚Üí \%resp    # opts: codec, row_group
Parquet::from_csv   $src, $dst, %opts ‚Üí \%resp    # CSV ‚Üí parquet; opts: header, delimiter, codec
Parquet::from_json  $src, $dst, %opts ‚Üí \%resp    # NDJSON ‚Üí parquet; opts: codec
Parquet::write      \@rows, $dst, %opts ‚Üí \%resp  # in-memory rows (hashrefs) ‚Üí parquet; opts: codec
Parquet::write_partitioned \@rows, $dst, $column, %opts ‚Üí \%resp  # Hive col=val/ dirs; opts: codec
Parquet::merge      \@srcs, $dst, %opts ‚Üí \%resp  # concat same-schema files (vertical); opts: codec
Parquet::hstack     $path, $other, $dst, %opts ‚Üí \%resp  # horizontal: append $other's columns at matching row count; column-name collision errors
Parquet::select     $path, $dst, \@cols, %opts ‚Üí \%resp  # project a column subset into a new file (column pruning); unknown column errors
Parquet::drop       $path, $dst, \@cols, %opts ‚Üí \%resp  # complement of select: keep all columns but \@cols; unknown column / drop-all errors
Parquet::rename     $path, $dst, \%map, %opts ‚Üí \%resp   # relabel columns { old => new }; preserves types/order/rows; unknown column / name-collision errors
Parquet::repartition $src, $dst, %opts ‚Üí \%resp   # rewrite with a target max row-group row count; opts: row_group (default 65536), codec
Parquet::to_ndjson  $path, $dst ‚Üí \%resp          # parquet ‚Üí NDJSON file (inverse of from_json)
Parquet::dtypes     $path ‚Üí \%resp                # each column's Arrow logical type: { num_fields, columns:[{name, dtype, nullable}] }
Parquet::metadata   $path ‚Üí \%resp                # writer kv metadata + created_by + version
```

### Row predicates / reshaping (polars-style)

Materialize the file once, then transform the JSON rows. Comparison is numeric
for numbers, lexicographic for strings; nulls always sort last.

```stryke
Parquet::filter      $path, $column, $op, %opts ‚Üí @rows   # rows where $column OP $value (eq/ne/gt/ge/lt/le, = != > >= < <=, is_null/is_not_null); opts: value, columns
Parquet::where_count $path, $column, $op, %opts ‚Üí $n      # count-only companion to filter (same grammar); opts: value
Parquet::filter_to   $path, $dst, $column, $op, %opts ‚Üí \%resp  # write only matching rows to a new file; empty result errors; opts: value, codec
Parquet::distinct    $path, %opts ‚Üí @rows                 # unique rows (polars unique); opts: columns (key + projection); first occurrence wins
Parquet::sort        $path, $column, %opts ‚Üí @rows        # ORDER BY $column; opts: descending => 1, columns; nulls last, stable
Parquet::column      $path, $column ‚Üí @values             # a single column's cells as a flat list (polars Series)
Parquet::sum         $path, $column ‚Üí $sum                # numeric sum of $column (nulls/non-numeric skipped)
Parquet::describe    $path ‚Üí \%resp                       # per-column { count, null_count, min, max, mean, sum }; non-numeric ‚Üí null numerics
Parquet::group_by    $path, $by, %opts ‚Üí @groups          # GROUP BY $by ‚Üí { key, count, value }; opts: agg, func (count/sum/min/max/mean)
Parquet::random_sample $path, %opts ‚Üí @rows               # n random rows (reservoir, reproducible per seed); opts: n, seed, columns; keeps file order
```

### Diagnostics

```stryke
Parquet::validate            $path ‚Üí { ok, rows, row_groups } | { ok:false, stage, detail }
Parquet::column_chunk_stats  $path ‚Üí @{ {row_group, num_rows, columns:[{column, compression,
                                          encodings, compressed_size, uncompressed_size,
                                          num_values, min, max, null_count}]} }
Parquet::size_report         $path ‚Üí { total_compressed_size, total_uncompressed_size,
                                       compression_ratio, num_rows, bytes_per_row,
                                       columns:[{column, compressed_size, uncompressed_size,
                                                 compression_ratio}] }   # columns sorted by size desc
Parquet::null_summary        $path ‚Üí { num_rows, total_nulls,
                                       columns:[{column, null_count, null_fraction}] }   # null_count null = unknown
Parquet::encoding_summary    $path ‚Üí { columns:[{column, encodings, compression}] }   # footer-only physical-encoding rollup per column
Parquet::bloom_filter_summary $path ‚Üí { columns:[{column, has_bloom_filter, chunks_with_filter}], columns_with_bloom_filter, columns_total }   # footer-only bloom-filter presence per column
Parquet::sorting_columns_summary $path ‚Üí { row_groups:[{row_group, sorting_columns:[{column, column_idx, descending, nulls_first}]}], has_sorting_columns }   # footer-only declared sort order
Parquet::sample              $path, %opts ‚Üí @rows   # opts: offset, n, columns ‚ÄĒ arbitrary window
Parquet::features            $path ‚Üí { has_bloom_filter, has_column_index, has_offset_index,
                                       columns:[{column, bloom_filter, column_index, offset_index}] }
```

`validate` reads every row group and reports failure as data (it never
`die`s on a corrupt file ‚ÄĒ check `ok`). `column_chunk_stats`, `size_report`,
and `features` read only the footer ‚ÄĒ `size_report` rolls the per-chunk byte
sizes up to file and per-column compression totals; `sample` fills the window
`head`/`tail` can't express.

### Convenience composites

Pure-stryke helpers over `schema` / `count` ‚ÄĒ no extra file reads beyond
the call they wrap.

```stryke
Parquet::column_names $path ‚Üí \@names      # schema field names, in file order
Parquet::column_count $path ‚Üí $n           # number of columns
Parquet::is_empty     $path ‚Üí 1 | 0        # count == 0 (schema may still exist)
```

### `stats` shape (one NDJSON line per column)

```json
{
  "name": "score",
  "type": "float64",
  "nullable": true,
  "null_count": 1247,
  "distinct_count": null,
  "min": -3.14,
  "max": 99.5
}
```

Min/max/null_count come from the footer per row-group statistics
aggregated across the whole file. `distinct_count` only fills in when the
writer wrote it (most parquet writers don't).

## [0x06] Supported compression codecs

| Codec | Library | Notes |
|---|---|---|
| `snappy` | `snap` | parquet-rs default; fast, modest ratio |
| `zstd` | `zstd` | best ratio per CPU |
| `gzip` | `flate2` | broad compatibility |
| `lz4` | `lz4_flex` | LZ4_RAW frame |
| `brotli` | `brotli` | high ratio, slow |
| `uncompressed` | ‚ÄĒ | fastest write, biggest file |
